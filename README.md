# SocialLLM

> Do LLMs develop internal representations that identify *who* they are talking to?

This project builds a social interaction framework where GPT-2-XL (the **target**) interacts with several other LLM agents on structured collaboration tasks, then uses **neuroscience-style single-unit and population analysis** to test whether the target's hidden-unit activations carry identity-specific signals about its interlocutor.

---

## Research question

When GPT-2-XL reads a message generated by a specific other LLM, do particular neurons (or neural populations) activate selectively — analogous to face-selective cells in inferotemporal cortex, or concept cells in the human MTL?

---

## Pipeline

```
1. Social interaction collection
   (run_interactions.py)
        │
        ▼
   [data/interactions/*.json]   ← conversation transcripts
   [data/activations/*.h5]      ← per-turn hidden-unit vectors
        │
        ▼
2. Neuroscience-style analysis
   (analyze.py)
        │
        ├── ANOVA F-stats  (which units vary across conditions?)
        ├── Selectivity Index  (how tuned is each unit?)
        ├── Lifetime sparseness
        ├── Tuning curves  (mean activation per condition)
        ├── Population decoding  (linear classifier across all units)
        ├── PCA / UMAP geometry
        └── RSA / RDMs
```

## Running

### Step 1 — Collect interactions & activations

```bash
python experiments/run_interactions.py \
    --config config/config.yaml \
    --device cuda \
    --n_episodes 10
```

This runs all (interlocutor × task) combinations and saves:
- `data/interactions/<id>.json` — conversation transcripts
- `data/activations/activations.h5` — activation dataset

### Step 2 — Analyse

```bash
python experiments/analyze.py \
    --config config/config.yaml
```

Outputs:
- `results/selectivity_summary.json` — per-layer scalar metrics
- `results/selectivity_reports.pkl` — full per-unit arrays
- `results/decoding_results.json` — cross-validated accuracy per layer
- `results/figures/*.png` — plots

### Interactive exploration

```bash
cd notebooks/
jupyter notebook
# open exploration.py via jupytext or run sections manually
```

## Interlocutor agents

| Agent ID | Model | Persona |
|---|---|---|
| `gpt2_small_curious` | GPT-2 (117M) | Curious, question-asking |
| `gpt2_medium_formal` | GPT-2-Medium (345M) | Formal, bullet-point answers |
| `gpt2_large_creative` | GPT-2-Large (774M) | Creative storyteller |
| `gpt2_small_concise` | GPT-2 (117M) | Terse, one-sentence answers |

Each agent has a distinct model size and persona, creating a distinct distribution of generated text — the question is whether GPT-2-XL's neurons encode this distinctness.

## Key analysis concepts

**Selectivity Index (SI)**  
`SI = (R_max - R_second_max) / (R_max + R_second_max)`  
Adapted from face-cell literature. SI = 1 means the unit responds to exactly one interlocutor.

**ANOVA + FDR**  
One-way ANOVA across conditions per unit; Benjamini-Hochberg FDR correction.

**Population decoding**  
Logistic regression trained on all units of one layer; cross-validated accuracy tells us how much *interlocutor identity* information is linearly decodable from the population vector.

**RSA**  
Representational Dissimilarity Matrices reveal how the geometry of interlocutor representations changes across layers.
